# Engineering

## Purpose

Defines engineering memory principles. Remember mistakes to avoid repetition.

## Memory Principles

Agents must remember:

| Memory Type | Content |
|-------------|---------|
| Failures | What caused failures |
| Complexity | What caused complexity |
| Instability | What caused instability |

## Core Reality

> "Good engineering comes from remembering mistakes."

## Memory Rules

| # | Rule |
|---|------|
| 1 | Prefer solutions that worked before |
| 2 | Avoid patterns that failed before |
| 3 | Repeat proven approaches |
| 4 | Avoid experimental redesign unless required |

## Pattern Retention

### PROVEN - Use Again

| Pattern | Use When |
|---------|----------|
| Simple solutions | Standard problems |
| Minimal dependencies | Any new code |
| Existing patterns | Consistency |
| Conservative approach | Production systems |

### FAILED - Avoid

| Pattern | Avoid Because |
|---------|---------------|
| Over-abstraction | Maintenance burden |
| Premature optimization | Complexity |
| Future proofing | Speculation |
| Clever code | Unmaintainable |

## Engineering Evolution

> "Engineering evolution comes from accumulated stability decisions."

### Evolution vs Revolution

| Revolution | Evolution |
|------------|-----------|
| Breaking changes | Incremental improvement |
| New paradigms | Proven patterns |
| Risk | Stability |
| Uncertainty | Knowledge |
